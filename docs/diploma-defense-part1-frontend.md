## Текст выступления

Здравствуйте. Я расскажу про **клиентскую часть** нашей платформы — то, что видит пользователь в браузере.

Мы сделали **gcloude** — это веб-кабинет, через который можно зарегистрироваться, создать проект, указать Git-репозиторий и задеплоить приложение, не заходя на сервер по SSH. Всё управление — через обычный сайт.

---

**[Открыть `https://app.kostricyn.ru`]**

Вот наш кабинет. Если зайти первый раз, внизу появляется **баннер про cookie** — это осознанное согласие пользователя: можно принять только необходимые cookie для входа или все. Тексты политик лежат отдельно — пользовательское соглашение, конфиденциальность, обработка персональных данных. Это важно с точки зрения работы с данными: человек не просто «кликает куда-то», а явно подтверждает условия.

---

**[Перейти на `/login`, если не авторизованы]**

Экран входа и регистрации мы специально упростили. Для входа — **email и пароль**. Для регистрации — имя, email, пароль и название **рабочего пространства** — это то, как команда будет называться в кабинете. Юрлицо и ИНН при регистрации не спрашиваем — они понадобятся позже, в биллинге.

Обязательная галочка: пользователь принимает **соглашение** и **политику обработки данных**. Без неё кнопка не активна. После успешного входа сессия сохраняется в cookie — пользователю не нужно логиниться при каждом обновлении страницы.

**[Войти / показать уже залогиненный кабинет]**

---

**[Дашборд — `/dashboard`]**

После входа попадаем на **обзор**. Здесь сводка: сколько проектов, приложений, последние деплои. Это точка входа — сразу видно, что происходит на платформе, без копания в настройках.

Слева — **боковое меню**. Мы сгруппировали разделы по смыслу:

- **Платформа** — обзор, проекты, приложения;
- **Релизы** — деплои, домены, логи;
- **Биллинг** — счета и документы;
- **Аккаунт** — команда и настройки.

Навигация с иконками, активный пункт подсвечивается. Вверху sidebar — название рабочего пространства того, кто вошёл.

---

**[Проекты — `/projects`]**

**Проект** — это логическая группа: например, один продукт или один клиент. Внутри проекта живут приложения.

**[Открыть карточку проекта]**

На странице проекта настраивается **деплой из Git**:

- ссылка на репозиторий;
- ветка, обычно `main`;
- можно задеплоить **frontend** и **backend** из одного монорепозитория — у каждого свой каталог и runtime: Node.js или Python.

Либо создаётся **отдельное приложение** — одна ссылка на Git, один runtime. Пользователь нажимает «Задеплоить» — дальше уже работает серверная часть, про неё расскажет мой коллега. С фронта мы только отправляем запрос и показываем статус.

---

**[Приложения — `/applications`]**

Здесь список **всех задеплоенных сервисов**: имя, runtime — node или python, статус.

Статусы понятные:

- **building** — идёт сборка;
- **running** — работает;
- **error** — что-то пошло не так.

**[Открыть карточку приложения]**

В карточке — **ссылка на живой сайт** вида `имя.apps.kostricyn.ru`, логи контейнера, кнопки остановить, перезапустить, передеплоить, удалить. Пользователь всё делает из браузера — не нужен терминал на VPS.

---

**[Кратко: Деплои, Настройки]**

В разделе **«Деплои»** — история сборок: из какого Git, когда, успех или ошибка.

В **«Настройки»** — вкладки: профиль, рабочее пространство, юрлицо для счетов, **домены** кабинета и приложений — `app`, `api`, `apps` на нашем домене.

**«Команда»** — участники и роли: кто может деплоить, кто смотрит биллинг.

## Про технологии фронта — коротко, если спросят

Если нужно одним абзацем про стек:

> Клиент написан на **Nuxt 4** и **Vue 3** с **TypeScript**. Nuxt даёт маршрутизацию — каждый URL это файл в папке `pages`. Общие компоненты — кнопки, карточки, поля — вынесены в `shared`, экраны проектов и приложений — в `common`.  
>  
> Кабинет **не собирает Docker и не трогает nginx** — только шлёт HTTP-запросы на `/api/v1/...`. Ответ приходит JSON, интерфейс его рисует. Запросы идут на тот же домен `app.kostricyn.ru`, nginx проксирует их на backend — пользователю не нужно знать внутренние порты.  
>  
> Защита маршрутов — middleware: без cookie-сессии в `/dashboard` и дальше не пустит.

## Если спросят только про фронт

| Вопрос | Короткий ответ |
|--------|----------------|
| Почему Nuxt, а не React? | Nuxt 4 + Vue 3 — file-based routing, SSR при необходимости, быстрая сборка кабинета |
| Где хранится авторизация? | Cookie `platform_session`, 7 дней, Secure на HTTPS |
| Фронт знает про Docker? | Нет, только API: «создай приложение», «запусти деплой» |
| Как обновляется статус сборки? | Polling при `building`, потом данные с `/api/v1/hosting/apps` |
| Адаптивность? | Layout responsive, sidebar + контент на desktop |
